# TG Bot with Memory

Telegram-бот с памятью на **aiogram 3** + **OpenAI** + **ChromaDB**.

В репозитории три версии бота — от простой к полной. Так удобнее учиться и сравнивать подходы к памяти.

---

## About

Проект показывает, как дать Telegram-боту «память» трёх типов:

| Бот | Файл | Что умеет |
|---|---|---|
| Короткая память | `bot_short_memory.py` | Помнит последние 10 сообщений диалога |
| Долгая память | `bot_long_memory.py` | Читает PDF/TXT/DOCX, хранит в векторной БД, отвечает по документу |
| Комбинированная | `bot.py` | И диалог, и документы одновременно |

**Зачем три бота**
- Понять разницу между history buffer и RAG.
- Отлаживать каждый тип памяти отдельно.
- В итоге собрать продакшен-вариант с обеими памятями.

**Преимущества комбинированного подхода**
- Короткая память → естественный диалог («а что ты сказал раньше?»).
- Долгая память → точные ответы по файлам и базе знаний компании.
- Вместе → бот и разговаривает в контексте, и опирается на документы, не выдумывая факты.

**Свойства**
- Короткая: быстро, в RAM, сбрасывается при перезапуске.
- Долгая: persistent (папка `./memory`), переживает рестарт.
- Комбо: system + RAG-контекст + история → Chat Completions.

## Быстрый старт

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполните BOT_TOKEN и OPENAI_API_KEY
python bot.py          # комбинированный бот
```

Отдельные версии:
```bash
python bot_short_memory.py
python bot_long_memory.py
```

Не запускайте два бота с одним токеном одновременно.

## Как устроена память

### Короткая память
- `dict[user_id] → deque` (последние 10 сообщений)
- Роли: `user` / `assistant`
- Сброс: `/start`, `/clear`

### Долгая память (RAG)
- Документ → чанки (~500 символов, overlap 50) → embeddings → **ChromaDB**
- Поиск релевантных фрагментов по вопросу
- Ответ модели строго с опорой на найденный контекст
- В `bot.py` при старте индексируется `company.txt`
